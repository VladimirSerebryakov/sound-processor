# Sound Processor (C++)

Консольное приложение на C++ для обработки и генерации звуковых файлов в формате **WAV** (16-bit PCM, 44100 Hz). Проект построен по принципам ООП и модульной архитектуры, позволяя легко расширять список доступных аудиоэффектов без изменения ядра системы.

---

## Особенности архитектуры
* **Фабрика фильтров**: Связывание строковых аргументов с объектами фильтров через реестр создателей (`std::map`).
* **Потоковая обработка**: Минимизация потребления памяти при работе с большими аудиофайлами за счет последовательного чтения и модификации блоков сэмплов в контейнерах `std::vector<int16_t>`.
* **Тестирование**: Полное покрытие ключевых модулей (математика, парсер аргументов, пайплайн, фильтры) модульными тестами на фреймворке **Catch2**.
* **Качество кода**: Кодовая база соответствует строгим правилам `clang-format` и `clang-tidy`.
* **Безопасность**: Код аккуратно обрабатывает все испорченные входные данные, выстроена система исключений.

---

## Описание архитектуры

Архитектура программы разделена на 4 понятных шага, которые работают друг за другом:

1. **Разбор команд**: За это отвечает `ArgsParser`. Он смотрит на то, что вы ввели в консоли, проверяет имена файлов и собирает список эффектов, которые вы хотите применить.
2. **Фабрика фильтров**: Класс `CmdLineArgs2PipelineConverter` с помощью контейнера `std::map` ставит в соответствие названию фильтра его продюссор - функцию для создания объекта фильтра.
3. **Конвейер обработки**: Класс `Pipeline` — это менеджер, который собирает все созданные фильтры в очередь и по очереди запускает их. Сам звук хранится в удобной обертке `Waveform` (внутри это обычный массив чисел `std::vector<int16_t>`).
4. **Чтение и запись**: Модули `WAVReader` и `WAVWriter` открывают аудиофайл, проверяют, что он не испорчен, переводят его в понятные программе числа для обработки, а в конце бережно сохраняют результат обратно на жесткий диск.

## Реализованные фильтры и генераторы

### 1. Звуковые эффекты (Эффекты обработки)
* **`ampl` (Amplifier)** — Изменение громкости звука на заданный коэффициент.
* **`silence` (Silence)** — Зануление выбранного промежутка времени в аудиопотоке. Поддерживает три режима указания интервала: в секундах (`sec`), миллисекундах (`ms`) или напрямую в сэмплах (`samples`).
* **`normalize` (Normalize)** — Нормализация звука до заданного пикового значения (от `0.0` до `1.0`).
* **`lowpass` (Lowpass Filter)** — Сглаживание скользящим окном с настраиваемым размером окна.
* **`timestretch` (Timestretch)** — Изменение скорости воспроизведения аудиодорожки методом линейной интерполяции сэмплов.

### 2. Синтезаторы (Генераторы звука)
* **`singen` (Sine Generator)** — Генерация чистого тона заданной частоты на основе синусоиды.
* **`amgen` (AM Generator)** — Генератор амплитудно-модулированного сигнала.
* **`fmfilter` (FM Generator)** — Частотно-модулированный синтезатор звука.
